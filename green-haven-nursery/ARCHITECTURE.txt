╔══════════════════════════════════════════════════════════════════════════════╗
║                              DATA FLOW EXAMPLE                               ║
║                      (User Placing an Order)                                 ║
╚══════════════════════════════════════════════════════════════════════════════╝

1. User fills checkout form
   ↓
2. Frontend: POST /api/checkout with delivery address
   ↓
3. Backend: Validates JWT token
   ↓
4. Backend: Retrieves user's cart from database
   ↓
5. Backend: Validates all products in stock
   ↓
6. Backend: Creates order record
   ↓
7. Backend: Generates unique delivery code (GH-XXXXXXXX)
   ↓
8. Backend: Creates order items from cart items
   ↓
9. Backend: Reduces product stock levels
   ↓
10. Backend: Creates payment record (COD)
    ↓
11. Backend: Clears user's cart
    ↓
12. Backend: Returns order with delivery code
    ↓
13. Frontend: Shows success modal with delivery code
    ↓
14. Frontend: Redirects to home page

║                            SECURITY LAYERS                                   ║
╚══════════════════════════════════════════════════════════════════════════════╝

┌─────────────────────────────────────────────────────────────────────────────┐
│ Layer 1: Frontend Validation                                                │
│ • Email format validation                                                   │
│ • Password length check (6+ chars)                                          │
│ • Phone number format                                                       │
│ • Required field validation                                                 │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│ Layer 2: HTTPS Transport                                                    │
│ • All data encrypted in transit                                             │
│ • JWT tokens sent in Authorization header                                   │
│ • CORS protection prevents unauthorized origins                             │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│ Layer 3: Backend Authentication                                             │
│ • JWT token validation on every request                                     │
│ • Token expiration (7 days)                                                 │
│ • User identity verification                                                │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│ Layer 4: Backend Validation                                                 │
│ • Input sanitization                                                        │
│ • SQL injection prevention (SQLAlchemy ORM)                                 │
│ • XSS protection                                                            │
│ • Business logic validation                                                 │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│ Layer 5: Database Security                                                  │
│ • Passwords hashed with bcrypt (never stored plain)                         │
│ • Foreign key constraints                                                   │
│ • Parameterized queries                                                     │
│ • Access control (user can only access own data)                            │
└─────────────────────────────────────────────────────────────────────────────┘
